2. Enhanced Naming Format

{DocumentType}-{SequentialNumber}-{ObjectName}[-{ColumnName}]-{JiraNumber}

Components:

Component Format Description Required
DocumentType 2 uppercase letters Identifies the type of document (BR, EN, DF, etc.) REQUIRED
SequentialNumber 4 digits (zero-padded) Auto-incremented counter per document type REQUIRED
ObjectName NEW Table/USP name The database object affected (without schema prefix) REQUIRED
ColumnName NEW Column name Specific column changed (omit for object-level changes) OPTIONAL
JiraNumber {Project}-{Number} Jira ticket reference (e.g., BAS-123) OPTIONAL

Enhanced Examples:

Column-Specific Enhancement:
EN-0001-irf_policy-PolicyNumber-BAS-123
Meaning: Enhancement #1 to the PolicyNumber column in irf_policy table, tracked in Jira BAS-123

Column-Specific Defect Fix:
DF-0002-irf_claim-ClaimAmount-BAS-456
Meaning: Defect Fix #2 to the ClaimAmount column in irf_claim table, tracked in Jira BAS-456

Table-Level Change:
EN-0003-irf_policy-BAS-789
Meaning: Enhancement #3 to the entire irf_policy table (no specific column), tracked in Jira BAS-789

Stored Procedure Enhancement:
BR-0001-usp_LoadPolicy-BAS-999
Meaning: Business Request #1 for the usp_LoadPolicy stored procedure, tracked in Jira BAS-999

3. Object and Column Naming Rules

ObjectName Format:

  • Tables: Use table name without schema (e.g., irf_policy not gwpc.irf_policy)
  • Stored Procedures: Use full procedure name including prefix (e.g., usp_LoadPolicy)
  • Views: Use view name without schema (e.g., vw_PolicySummary)

When to Include ColumnName:

Scenario Include Column? Example
Change affects a specific column YES EN-0001-irf_policy-PolicyNumber-BAS-123
Change affects multiple columns NO EN-0002-irf_policy-BAS-456
Change affects entire table NO BR-0003-irf_claim-BAS-789
Change is to stored procedure NO DF-0004-usp_LoadPolicy-BAS-999

5. Search and Retrieval Benefits

Enhanced Search Capabilities:

Search For Pattern Example Results
All changes to a specific table *-{TableName}-* *-irf_policy-* All changes to irf_policy table
All changes to a specific column *-{TableName}-{ColumnName}-* *-irf_policy-PolicyNumber-* All changes to PolicyNumber in irf_policy
All enhancements to a table EN-*-{TableName}-* EN-*-irf_policy-* All enhancements to irf_policy
All defect fixes for a stored procedure DF-*-{ProcedureName}-* DF-*-usp_LoadPolicy-* All defect fixes to usp_LoadPolicy

Database Query Examples:

Find all changes to irf_policy table: 
SELECT SourceDocumentID, DocumentTitle, ColumnName, DocumentType
FROM DaQa.MasterIndex
WHERE TableName = 'irf_policy'
ORDER BY CreatedDate DESC;
Find all column-specific changes: 
SELECT SourceDocumentID, TableName, ColumnName, DocumentType
FROM DaQa.MasterIndex
WHERE ColumnName IS NOT NULL
ORDER BY TableName, ColumnName;
Find all enhancements to PolicyNumber column: 
SELECT SourceDocumentID, DocumentTitle, GeneratedDocURL
FROM DaQa.MasterIndex
WHERE TableName = 'irf_policy'
  AND ColumnName = 'PolicyNumber'
  AND DocumentType = 'Enhancement';

7. Best Practices with Enhanced Format

‚úÖ DO:
  • Always include object name in DocId
  • Include column name for column-specific changes
  • Use exact object/column names from database (case-sensitive)
  • Keep object names without schema prefix
  • Verify object/column exist in database before generating DocId
‚ùå DON'T:
  • Don't include schema in object name (use irf_policy, not gwpc.irf_policy)
  • Don't include column name for object-level or multi-column changes
  • Don't use abbreviated or shortened object names
  • Don't change object/column spelling from database
